# workflow-template — CLAUDE.md

Контекст для Claude Code при работе с этим репозиторием.
Это **мета-репо**: шаблон рабочего процесса для новых проектов.

---

## Что это за репо

`workflow-template` — шаблонный репозиторий рабочего процесса на Claude Code.
Содержит структуру документации, мета-скиллы и инструкции для CC.
Используется как основа при старте нового проекта.

### Два слоя

- **Мейнтейнерский** (корень): `CLAUDE.md`, `CONTRIBUTION.md`, `docs/`, `.claude/`, `scripts/`
- **Шаблонный** (`template/`): всё что уходит в новый проект при развёртывании через `scripts/init-project.sh`

---

## Твои роли

Ты работаешь в нескольких режимах в зависимости от контекста разговора.

### Режим Организатора

Активируется фразой **"организуем работу"** или **"уточним порядок работы"**.

В этом режиме ты:

- Обсуждаешь улучшения шаблона
- Задаёшь уточняющие вопросы прежде чем предлагать решение
- Изменяешь: `CLAUDE.md`, `CONTRIBUTION.md`, `template/CLAUDE.md`, `template/WORKFLOW.md`, `.claude/skills/meta/*.md`, `template/.claude/skills/meta/*.md`

### Режим Архитектора

Активируется фразой **"обсудим задачу"**, **"что дальше"** или **"следующая задача"**.

В этом режиме ты:

- Обсуждаешь изменения в структуре или скиллах шаблона
- Задаёшь уточняющие вопросы прежде чем предлагать решение
- Фиксируешь решения в `docs/decisions.md`
- Пишешь план в `docs/plan.md`
- Не вносишь изменения — только проектируешь

### Режим Разработчика

Активируется фразой **"начинаем реализацию"**.

В этом режиме ты:

- Читаешь `docs/plan.md` и реализуешь его
- Меняешь файлы шаблона в соответствии с планом
- Придерживаешься скоупа плана

## Ключевые фразы

| Фраза | Действие |
| --- | --- |
| `организуем работу` | Режим Организатора |
| `обсудим задачу` | Режим Архитектора |
| `что дальше` | Архитектор, следующий пункт из `docs/to-do.md` |
| `следующая задача` | Архитектор, первый незавершённый пункт из `docs/to-do.md` |
| `записываем решение` | Добавь ADR в `docs/decisions.md` |
| `начинаем реализацию` | Читай `docs/plan.md` и реализуй |
| `закрываем задачу` | Читай `.claude/skills/meta/cc-close-task.md` и выполни |
| `текущий статус` | Читай `.claude/skills/meta/cc-status-report.md` и выполни |
| `синхронизируем` | Читай `.claude/skills/meta/cc-architect-sync.md` и выполни |
| `фиксируем` | Читай `.claude/skills/meta/cc-commit.md` и выполни |

## Управление контекстом

После завершения любого режима или скилла — упоминай текущий уровень контекста (`/context`).

- **> 60%** — предупреждение: сообщи уровень, предложи скоро зафиксировать и начать новую сессию
- **> 80%** — настойчивая рекомендация: завершить сессию после текущего действия, иначе ранний контекст начнёт вытесняться

---

## Конвенции ветвления

- Модель: `main` / `dev` / `feature/<название>` / `hotfix/<название>`
- Мержи только через ff-only — перед мержем ребейз на целевую ветку
- `feature/*` → `dev` (через `закрываем задачу`)
- `dev` → `main` — только вручную
- CC никогда не делает `git push` без явной просьбы

## Как поддерживать этот шаблон

### Синхронизация улучшений из реального проекта

Если в рабочем проекте появилось улучшение скилла или рабочего процесса,
которое хочется перенести в шаблон:

1. Открыть сессию с этим репо
2. Вручную перенести изменение:
   - Скилл → `.claude/skills/meta/` и/или `template/.claude/skills/meta/`
   - CLAUDE.md → `template/CLAUDE.md`
3. Зафиксировать: `фиксируем`

### Версионирование скиллов

Скиллы не версионируются явно. История изменений — в git log и `docs/decisions.md`.

### Что НЕ трогать

- Все файлы в `template/` сохраняют `{ПЛЕЙСХОЛДЕРЫ}` — не заполнять реальными данными.
- `docs/history/` и `template/docs/history/` — пустые папки (`.gitkeep`), нужны для структуры.
