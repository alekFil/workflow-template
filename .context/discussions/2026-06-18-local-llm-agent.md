# Обсуждение: минимальный агент на локальной LLM

**Дата:** 2026-06-18
**Статус:** обсуждение завершено, реализация не начата

---

## Контекст

Идея: реализовать такой же подход к управлению проектом (ключевые фразы, роли, скиллы),
как в workflow-template, но с использованием локальной модели Qwen3-30B-FP8,
запущенной на сервере через vLLM. Интерфейс — CLI.

---

## Принятые решения

### Стек

- **Pydantic AI** — фреймворк агента (автор знаком с ним)
- **vLLM** — сервер модели, отдаёт OpenAI-совместимый API
- **Qwen3-30B-FP8** — локальная модель; поддерживает tool use нативно
- **Typer** — CLI-обёртка, REPL-цикл

Подключение: `OpenAIModel(base_url="http://...", api_key="EMPTY")` — стандартный паттерн для vLLM.

### Персистентность истории

Не нужна. Каждый запуск агента — чистый контекст.
Нужный контекст собирается заново из файлов `.context/` и скиллов перед каждым вызовом.

### Роутер ключевых фраз

Ключевая фраза распознаётся **до** вызова LLM — простым словарём в `router.py`.
Это надёжнее, чем просить модель самостоятельно определять режим.

Роутер возвращает:
- режим (Организатор / Архитектор / Разработчик)
- путь к скилл-файлу (или `None`)

### Сборка системного промпта

`loader.py` собирает системный промпт перед каждым вызовом:
`CLAUDE.md` (эквивалент) + скилл-файл режима (если есть).
Скиллы хранятся в `.claude/skills/meta/*.md` — те же файлы, что и сейчас.

### Инструменты агента

Минимальный набор через `tools.py`:
- `read_file` — чтение `.context/`, скиллов
- `write_file` — запись в `.context/decisions.md`, `.context/plan.md` и т.д.
- `run_git` — для скилла `cc-commit`

Qwen3 поддерживает tool use, инструменты работают нативно через Pydantic AI.

---

## Предложенная структура

```text
agent/
  cli.py          ← точка входа (typer), REPL-цикл
  router.py       ← ключевые фразы → режим + путь к скиллу
  agent.py        ← Pydantic AI агент, vLLM как OpenAI-provider
  tools.py        ← инструменты: read_file, write_file, run_git
  session.py      ← история сообщений (in-memory, живёт до выхода из CLI)
  loader.py       ← читает CLAUDE.md, .context/*.md, skills/*.md
```

### Поток вызова

```text
user input
  → router.py    распознаёт ключевую фразу, выбирает режим
  → loader.py    собирает system prompt = CLAUDE.md + скилл-файл (если есть)
  → agent.py     PydanticAI Agent(model=OpenAIModel(base_url=vllm))
  → tools.py     агент вызывает инструменты если нужно
  → CLI output
```
