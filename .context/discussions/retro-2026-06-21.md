# Ретроспектива 2026-06-21

## Что шло хорошо

- ADR-014–018 реализованы без разворотов — OSS-публикация была крупной задачей (перевод двух слоёв, слэш-команды, скрипты), но ни одно из решений не пересматривалось в процессе
- Проактивное обнаружение конфликта `/status` (ADR-017) — конфликт со встроенной командой CC был замечен и устранён до того, как стал проблемой для пользователей
- Дисциплина области задач — `template-mini` (ADR-010) и Cookiecutter (ADR-003) отклонены без реализации, что сберегло ресурс на осмысленные задачи
- Двухслойная архитектура (ADR-001) не потребовала пересмотра — фундаментальное решение держится стабильно на протяжении всей истории проекта
- Языковая стратегия (`Respond in the user's language`) — простое и элегантное решение, делающее шаблон языково-нейтральным без усложнения

## Что было трудно

- install.sh — три итерации исправлений: `fix: исправить интерактивные read при curl | bash` → `fix: устранить точки трения для нового пользователя` → `fix: handle inaccessible /dev/tty in install.sh` — интерактивный shell в curl-пайпе оказался постоянным источником проблем
- Демо не начато — Priority 1 из ADR-014, остаётся незакрытым несмотря на то что вся остальная OSS-задача выполнена; является блокером для публикации
- ADR-004 (управление decisions.md) в backlog с самого начала — задача запланирована, но ни разу не приоритизирована; файл decisions.md уже содержит ADR со статусом «Отклонено» и частично устаревшие записи

## Паттерны и наблюдения

- Документация предшествует реализации: почти каждая задача идёт по паттерну `docs: обновить план` → `feat:` — это последовательная практика, которая держится на протяжении всей истории
- Скриптовый код требует итераций: shell-скрипты для пользовательского инструмента без end-to-end теста порождают цикличные баги; тест install.sh уже в «техническом долге», но с низким приоритетом несоразмерно риску
- Слэш-команды в мейнтейнерском слое != слэш-команды в шаблоне: `/retro` добавлен в мейнтейнерский слой, но не синхронизирован с `template/` — паттерн расхождения слоёв, который периодически возникает
- Соотношение плановой и внеплановой работы: ~85% коммитов — плановые; внеплановые — только баги install.sh

## Действия

- Синхронизировать `/retro` в шаблонный слой (`template/.claude/commands/retro.md`, `template/.claude/skills/meta/cc-retrospective.md`, `template/CLAUDE.md`)
- Переместить «Тест install.sh end-to-end» из раздела «После MVP» в Приоритет 2
- ADR-019: зафиксировать решение добавить `/retro` как регулярный инструмент рабочего процесса
