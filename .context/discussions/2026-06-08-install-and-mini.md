# Обсуждение: curl-установка и mini-версия шаблона

**Дата:** 2026-06-08
**Статус:** завершено, зафиксировано в ADR-005 и plan.md

---

## Контекст

Обсуждение двух связанных вопросов:

1. Как улучшить процесс установки шаблона (curl вместо клона)
2. Нужна ли упрощённая mini-версия для сообщества

---

## Часть 1: curl-установка

### Исходная идея

Вместо `git clone` + `./scripts/init-project.sh` — инициализировать свой репозиторий
и скачать скрипт установки через curl:

```bash
mkdir my-project && cd my-project
git init
curl -fsSL https://raw.githubusercontent.com/<user>/workflow-template/main/scripts/install.sh | bash
```

### Ключевые решения

- Скачивать `template/` через GitHub tar.gz одним запросом (`--strip-components=2`)
- Репо публичное, версионирование не нужно — всегда `main`
- Оба скрипта остаются: `init-project.sh` (для мейнтейнера) + `install.sh` (для пользователей)
- ADR-003 (cookiecutter) откладывается — `install.sh` покрывает текущий сценарий проще

Зафиксировано в ADR-005.

## Часть 2: оценка репозитория для сообщества

### Сильные стороны

- Разграничение фаз Архитектор/Разработчик — нетривиальная и ценная идея
- ADR-интеграция с AI-рабочим процессом — встречается редко
- Управление контекстом — реальная проблема, которую мало кто решает системно

### Барьеры для сообщества

- Русский язык — главный блокер (GitHub-аудитория читает по-английски)
- Сложный вход: двухслойная структура, скиллы, скрипты — слишком много трения
- Триггерные фразы на русском непрозрачны для внешнего пользователя
- Нет English README с примером «до/после»

### Что нужно для принятия сообществом

1. English README с конкретным примером диалога (без шаблона vs с ним)
2. Минимальная версия: один файл CLAUDE.md, копируется за 30 секунд
3. Упрощение / замена триггерных фраз на что-то более прозрачное
4. Дистрибуция: awesome-claude-code, форум Anthropic, dev.to

## Часть 3: mini-версия

### Идея

Два артефакта из одного репо:

- **mini** — один файл `CLAUDE.md`, немедленная ценность, для новых пользователей
- **full** — текущая полная система, для тех кто зацепился

### Почему одно репо (не два)

Mini — подмножество full. Два репо → ручная синхронизация при каждом изменении концепции.
В одном репо: `template/` (full) + `template-mini/` (mini, один файл).

### Установка

```bash
# mini
curl -fsSL .../scripts/install-mini.sh | bash

# full
curl -fsSL .../scripts/install.sh | bash
```

`install-mini.sh` скачивает один файл (`CLAUDE.md`) — никакого tar, один запрос.

### Следующий шаг

Спроектировать содержимое `template-mini/CLAUDE.md` — без него остальное бессмысленно.
