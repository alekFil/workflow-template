# Патч: настройка Markdown-конвенций

Перенести в другой проект два изменения.

---

## 1. Файл `.markdownlint.json` (создать в корне проекта)

```json
{
  "MD004": { "style": "dash" },
  "MD013": false,
  "MD060": false,
  "MD022": true,
  "MD032": true,
  "MD035": { "style": "---" },
  "MD040": true,
  "MD036": true
}
```

Что настраивает:

| Правило | Настройка | Смысл |
|---|---|---|
| MD004 | dash | Маркеры списков только `-` |
| MD013 | false | Отключить проверку длины строки |
| MD022 | true | Пустые строки вокруг заголовков |
| MD032 | true | Пустая строка перед списком |
| MD035 | `---` | Разделители только через `---` |
| MD036 | true | Запрет bold как замена заголовка |
| MD040 | true | Всегда указывать язык в блоке кода |
| MD060 | false | Отключить (вспомогательное правило) |

---

## 2. Секция в CLAUDE.md (добавить перед разделом «Переменные окружения»)

```markdown
## Конвенции Markdown

- Маркеры списков: только `-` (не `*`, не `+`)
- Блоки кода: всегда указывать язык — `python`, `bash`, `yaml`, `json`, `text` для plain text
- Перед заголовком и после — пустая строка
- Перед списком — пустая строка (если предшествует текст)
- Не использовать **bold** как заменитель заголовка — только `##`, `###` и т.д.
- Разделители `---`: использовать редко, только между крупными разделами; предпочитать иерархию заголовков
```
