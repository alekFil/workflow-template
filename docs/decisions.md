## ADR-002: Перенос index.md из docs/ в .claude/

**Статус:** Принято

**Проблема:** `docs/index.md` — навигатор для Claude Code (что читать и когда), но лежит в `docs/` рядом с рабочими артефактами проекта. Это нарушает логику: `docs/` для документации проекта, `.claude/` для инструментария CC.

**Решение:** Переместить `docs/index.md` → `.claude/index.md`. В шаблонном слое соответственно `template/docs/index.md` → `template/.claude/index.md`.

**Последствия:**

- `.claude/` становится полным домом для CC-инструментария: настройки, скиллы, навигатор.
- Ссылки на `docs/index.md` в `CLAUDE.md` и `CLAUDE.template.md` обновляются на `.claude/index.md`.

## ADR-001: Разделение мейнтейнерского и шаблонного слоёв

**Статус:** Принято

**Проблема:** Репо одновременно является местом разработки шаблона и самим шаблоном. Файлы документации (`docs/status.md`, `docs/plan.md` и др.) использовались и как живые рабочие документы мейнтейнера, и как шаблонные заготовки с плейсхолдерами — это конфликт.

**Решение:** Разделить на два независимых слоя в одном репо:

- Мейнтейнерский слой — корень репо: `CLAUDE.md`, `CONTRIBUTION.md`, `SETUP.md`, `docs/`, `.claude/skills/meta/`, `scripts/`.
- Шаблонный слой — папка `template/`: `CLAUDE.md`, `WORKFLOW.md`, `template/docs/` (с плейсхолдерами), `template/.claude/skills/meta/`.

Развёртывание нового проекта — через скрипт `scripts/init-project.sh`: интерактивно заполняет плейсхолдеры, перемещает `template/` в корень, удаляет мейнтейнерский слой, переинициализирует git, привязывает новый remote, делает init commit.

Мейнтейнерская шпаргалка по работе с репо — `CONTRIBUTION.md` (не копируется в новый проект). Шпаргалка для нового проекта — `template/WORKFLOW.md`.

**Последствия:**

- `docs/` содержит реальную документацию мейнтейнера, скиллы (cc-status-report и др.) работают корректно.
- `.claude/skills/meta/` и `template/.claude/skills/meta/` — независимые копии, расходятся по мере развития.
- `CLAUDE.template.md` переименовывается в `template/CLAUDE.md`.
- `SETUP.md` остаётся в корне как инструкция по развёртыванию (удаляется скриптом).
