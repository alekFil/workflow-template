# Инструкция по настройке рабочего процесса

Этот шаблон переносит организацию работы с Claude Code в новый проект.

---

## Шаги настройки

### 1. Распакуй архив в корень нового проекта

```bash
tar -xzf workflow-template.tar.gz -C /path/to/your/project
```

### 2. Активируй CLAUDE.template.md

В этом репо два файла CLAUDE:

- `CLAUDE.md` — инструкции мейнтейнера шаблона (удали его)
- `CLAUDE.template.md` — заготовка для твоего проекта (переименуй)

```bash
rm CLAUDE.md
mv CLAUDE.template.md CLAUDE.md
```

### 3. Заполни плейсхолдеры в CLAUDE.md

Найди и замени все `{...}` в `CLAUDE.md`:

| Плейсхолдер | Что вставить |
| --- | --- |
| `{PROJECT_NAME}` | Название проекта |
| `{PROJECT_DESCRIPTION}` | 1–2 предложения о проекте |
| `{LAYER_1}` / `{TECH_1}` и т.д. | Строки таблицы стека |
| `{REPO_URL}` | Ссылка на репозиторий |
| `{ARCHITECTURE_RULES}` | Ключевые правила архитектуры |
| `{RUNTIME_SETUP}` | Команды окружения (uv/pip/npm/...) |
| `{CODE_CONVENTIONS}` | Конвенции кода |
| `{PROJECT_STRUCTURE}` | Дерево директорий |

### 4. Заполни docs/blueprint.md

Опиши реальную или планируемую архитектуру проекта.

### 5. Заполни docs/to-do.md

Внеси первые задачи в раздел "Следующее".

### 6. Настрой cc-export-chat.md (опционально)

Если нужен экспорт истории чата:

- Положи скрипт `export_chat_history.py` в `.claude/scripts/`
- Обнови `PROJECT_SLUG` в скилле `cc-export-chat.md`

### 7. Начни работу с CC

Скажи Claude Code:

```text
Прочитай CLAUDE.md и подготовь проект к работе. Помоги заполнить оставшиеся плейсхолдеры.
```

---

## Что входит в шаблон

```text
CLAUDE.md                          ← инструкции мейнтейнера (удалить при развёртывании)
CLAUDE.template.md                 ← заготовка CLAUDE.md для нового проекта (переименовать)
WORKFLOW.md                        ← шпаргалка по рабочему процессу
.claude/
  skills/
    meta/
      cc-commit.md                 ← скилл "фиксируем"
      cc-close-task.md             ← скилл "закрываем задачу"
      cc-status-report.md          ← скилл "текущий статус"
      cc-architect-sync.md         ← скилл "синхронизируем"
      cc-export-chat.md            ← скилл "экспорт истории"
docs/
  index.md                         ← карта документации
  blueprint.md                     ← архитектура проекта
  plan.md                          ← текущая задача (формат)
  to-do.md                         ← очередь задач
  status.md                        ← состояние реализации
  decisions.md                     ← ADR
  history/                         ← архив status.md (заполняется автоматически)
  discussions/                     ← обсуждения и исследования
```

---

## Если хочешь улучшить шаблон

Работай прямо в этом репо — здесь свой `CLAUDE.md` с инструкциями мейнтейнера.
Изменения в скиллах переносятся вручную из рабочих проектов.
